Overview

YADRO (Your Architecture's Definitive Runtime Optimizer) is a systems programming language designed for operating system development and low-level programming. Created by the CyrOil team, YADRO combines the safety guarantees of modern languages with the performance and control of low-level systems programming.

Core Philosophy & Memory Model

Dual-Level Memory Model

YADRO employs a sophisticated memory management approach:

  • Static Ownership: All types are unique by default. Assignment moves ownership.
  • Borrowing: Access via immutable (&T) or mutable (&mut T) references with compile-time lifetime checking.
  • Managed Heap: The gc<T> type provides reference-counted pointers for dynamic lifetimes.
  • Explicit Copying: Only #Copy types can be copied implicitly.

Key Principles

// Ownership - moves by default int x = 10 int y = x // x is moved, no longer accessible // Borrowing - references with lifetimes &int ref = &x &mut int mut_ref = &mut x // Managed heap for complex ownership gc[string] s = gc[string]("Hello, YADRO!") // Immediate destruction del x

The Four Technical Pillars

YADRO's design is built upon four foundational pillars that ensure safety, performance, and correctness:

Pillar 1

Ownership & Borrowing
Memory safety guaranteed by static single ownership with checked, scoped references.

Pillar 2

Protocol-Based Polymorphism
Generics constrained by explicit, compile-time-checked protocols for zero-cost abstraction.

Pillar 3

Algebraic Effects for Errors
Failure modeled using Result[T, E] type. Exceptions are banned.

Pillar 4

Region-Based Resource Management
arena construct provides deterministic, fast lifetime management.

YUPPI - Yadro User Project Package Index

YUPPI is the official package manager and distribution system for YADRO modules, libraries, and tools. It provides dependency resolution, version management, security verification, and seamless ecosystem integration.

Key Features

  • Automatic dependency resolution
  • Semantic versioning with compatibility checking
  • Cryptographic verification of package integrity
  • Cross-platform consistent package format
  • Seamless integration with YADRO compiler toolchain

Command Line Interface

yadro yuppi install <package> # Install package
yadro yuppi install network-lib --features ssl,compression
yadro yuppi add <package> # Add to dependencies
yadro yuppi build # Build current package
yadro yuppi publish # Publish to registry
yadro yuppi search <query> # Search packages

The YADRO Constitution

The Constitution establishes the foundational, immutable principles of the YADRO programming language. It serves as the supreme authority for all design decisions and evolution of the language.

Article I: Primacy of Static Verification

The correctness and safety of a program must be provable at compile time. The compiler is a verification engine, not just a translator.

Article II: Principle of Explicit Intent

Code must unambiguously express programmer intent. No magic, no hidden control flow. Cost must be apparent from syntax.

Article III: Zero-Cost Abstraction

Programmers can descend from high-level abstractions to low-level operations without losing performance or control.

Article IV: Determinism & Composability

System behavior must be predictable. Unmanaged global mutable state is prohibited. Side-effects are part of the type.

Standard Library Structure

YADRO's standard library is minimal, idiomatic, and demonstrates correct YADRO style:

std::core

Core types, traits, intrinsics

std::alloc

Memory allocators (global, arena)

std::collections

Data structures (vector, map, set)

std::os

OS interaction (fs, process, bios)

std::async

Async/await, actors, channels

std::libc_safe

Safe bindings to libc

Get Started with YADRO

Ready to dive in? Here's how to get started with YADRO development:

  1. Install YADRO: Download the compiler for your platform
  2. Initialize a project: yadro yuppi init my-project
  3. Write your code: Create src/main.yad
  4. Build: yadro yuppi build
  5. Run: Execute your compiled binary
